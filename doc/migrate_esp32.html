ESP32 Integrations-Leitfaden

Dieses Dokument beschreibt, wie das DCC-Decoder-Konfigurationsinterface in ein ESP32-Projekt (z.B. unter PlatformIO) integriert wird.

1. Architektur-Entscheidung

WICHTIG: Das Parsing der .decdb (XML) Dateien erfolgt vollständig im Browser (Client-side). Der ESP32 fungiert lediglich als statischer Webserver und als API-Gateway zur DCC-Hardware.

2. Dateisystem-Setup (LittleFS)

Der ESP32 sollte LittleFS verwenden. Die Dateien aus diesem Projekt werden in den data/ Ordner des PlatformIO-Projekts kopiert:

  • data/index.html
  • data/style.css
  • data/app.js
  • data/files.js (Liste der verfügbaren Decoder)
  • data/decoder/*.decdb (Die XML-Definitionen)
  • data/decoder/hersteller.json

3. Webserver-Endpunkte (ESP32 Seite)

Der ESP32 muss folgende Routen bereitstellen:

// Beispiel mit ESPAsyncWebServer
server.on("/api/read", HTTP_GET, [](AsyncWebServerRequest *request){
    int cv = request->getParam("cv")->value().toInt();
    // Hardware-Logik: DCC Lese-Befehl auslösen
    int value = dccMain.readCV(cv); 
    request->send(200, "text/plain", String(value));
});

server.on("/api/write", HTTP_POST, [](AsyncWebServerRequest *request){
    int cv = request->getParam("cv")->value().toInt();
    int val = request->getParam("val")->value().toInt();
    // Hardware-Logik: DCC Schreib-Befehl auslösen
    dccMain.writeCV(cv, val);
    request->send(200);
});

4. Anpassungen in app.js

In der app.js müssen die Dummy-Funktionen für Read/Write durch echte API-Calls ersetzt werden:

// In app.js (Beispiel für Read)
async function readCVFromHardware(cvNumber) {
    const response = await fetch(`/api/read?cv=${cvNumber}`);
    const value = await response.text();
    return parseInt(value);
}

// Beispiel für Write
async function writeCVToHardware(cvNumber, value) {
    await fetch(`/api/write?cv=${cvNumber}&val=${value}`, { method: 'POST' });
}

5. Optimierung für Flash-Speicher

Da .decdb Dateien groß sein können (z.B. >700KB), empfiehlt es sich:

  1. Nur die benötigten Decoder-Dateien in den Flash zu laden.
  2. Dateien mit GZIP zu komprimieren (LittleFS unterstützt .gz Dateien, die der Webserver automatisch erkennt).
